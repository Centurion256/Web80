HTML: Structure

---------------

We already mentioned that HTML uses elements to logically structure a page. But how do we do that? In order to speparate and distinguish elements, HTML uses tags. Tags basically tell the reader which element they're looking at, what properties does it have and where does it end. Tag is the very basic component that describes the web page, all content is in some tag or another. There are numerous tag for different purposes, we will only look at a few of them in this tutorial, but you can find an exhaustive list in the references.

--------------------------

My cat is very grumpy

In terms of HTML, this would mean that we have to represent it as a "paragraph" element. Luckily, HTML has a <p> tag. Elements may also have different attributes, what would distinguish them from other similar elements or make it easier to classify them. This will be further elaborated upon in our next section.


There are some other important properties, for exapmple we can nest elements inside other elements. Some elements also may be empty, and contain no other content but their name and attributes. A good example would be a <br> element, which simply breaks the line.


To conclude, writing good HTML is basically finding the correct tags to use in a given occasion. There is another important divison of HTML elements: Semantic and Non-semantic.

Semantic elements are elements which are clearly descriptive of their content, e.g. <p> always should be used to denote a paragraph, <br> denotes a break in the line, <img> contains an image, etc. Most HTML elements are semantic. Does this, however, imply that they are the most commonly used? Suprisingly, no.


Non-semantic elements are generic elements: they tell nothing about the content, just that there is some divison. Two most common non-semantic elements are <div> and <span>, the first (by default) used for block separation and the latter for inline. These two, <div> especially, are very commonly used. The reason is that while they tell nothing about the content explicitly, they do, combined with appropriate attributes, divide the page logically. This is especially useful for CSS and scripting languages, but more on that later.

Anatomy of HTML Document

------------------------

We now know what elements are and how to represent them in HTML. But what does the entire HTML page consist of?


The first line should ALWAYS be <!DOCTYPE html>. Nowadays, this doesn't really represent anything, but in the past it was used to represent a ruleset/convension that the document would follow. Nowadays, this is obsolete, but the line still remains. Always remember to contain it at the top of the document.


Next is the html element. It is known as the root element that encloses the content of your document.


The head element is used for content that describes the page itself, but isn't displayed to the end user. This doesn't mean that it doesn't affect the rest of the document: in fact it does, since such information as the title of the document, keywords for search engines, character encoding, links to external content that is used, such as CSS styles, fonts or external libraries, etc. We won't cover a lot of it in this tutorial, but more information is available in the references.


The body element is where most of the content of the document is located.
