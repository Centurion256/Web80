HTML: Interactive content

-------------------------

By now, if you have read all the previous articles, looked at the source code and opened the tag/attribute references, you know the very basics of HTML. In fact, you probably can recreate this website in it's entirety! (without, of course, the styles). But let's take a look at a few other important features of HTML


Modern HTML allows us to insert very advanced elements, which are essential to some of the biggest platforms on the web today: video and audio. MDN describes them perfectly:

Web developers have wanted to use video and audio on the Web for a long time, ever since the early 2000s when we started to have bandwidth fast enough to support any kind of video (video files are much larger than text or even images.) In the early days, native web technologies such as HTML didn't have the ability to embed video and audio on the Web, so proprietary (or plugin-based) technologies like Flash (and later, Silverlight) became popular for handling such content. This kind of technology worked ok, but it had a number of problems, including not working well with HTML/CSS features, security issues, and accessibility issues.


A native solution would solve much of this if implemented correctly. Fortunately, a few years later the HTML5 specification had such features added, with the <video> and <audio> elements, and some shiny new JavaScript APIs for controlling them. We'll not be looking at JavaScript here - just the basic foundations that can be achieved with HTML.

Video and Audio

---------------

Let's look at the elements themselves. The video element allows you to embed a video. It's notable attributes are:

  • src - specifies the URL/path to the video content
  • controls - allows user to control video playback
  • width and height - sets the width and height of the videoplayer
  • autoplay - plays the video automatically
  • loop - loops the video playback
The content inside the video tag is the so-called fallback content: it will be displayed in case the browser doesn't support video elements.


The audio element works similarly to video, except that it doesn't support the width and height attributes. It is also a good practice to add a child track element, in order to make the content more accessible by providing captions and a transcript. You can read more about the track element on MDN.
