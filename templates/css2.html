CSS is a rule-based language - you define rules specifying groups of styles that should be applied to particular elements or groups of elements on your web page.


For example, let's say that we want all main headings to be shown in red and their font size to be 10 pixels. In CSS, this will translate into the following rule

h1 { color: red; font-size: 10px; }

The rule opens with a selector. This selects the HTML element that we are going to style. In this case we are styling level one headings (<h1>).


We then have a set of curly braces { }. Inside those will be one or more declarations, which take the form of property and value pairs. Each pair specifies a property of the element(s) we are selecting, then a value that we'd like to give the property. Before the colon, we have the property, and after the colon, the value.


The property is an identifier, that is a human-readable name, that defines which feature is considered. The value describes how the feature must be handled by the engine. Each property has a set of valid values.


In our example, we have two properties: color and font-size, each set to a corresponding values, which are red and 10p(i)x(els), respectively. Essentially, a CSS stylesheet is just a collection a different rules for different selectors

Now we know what the basic structure of a CSS rule is, let's go deeper into the topic of selectors


In CSS, selectors are used to target the HTML elements on our web pages that we want to style. A CSS selector is the first part of a CSS Rule. There are different types of selectors, which allows us to style the desired elements in a desired way. In essence, writing good CSS is picking the right selectors for rules and using the correct attributes.


In our previous example, we used a h1 selector in our CSS rule. What this meant was that for every level-one heading on the page, the style would be as follows. So therefore, this rule would apply to every ELEMENT h1. Such selectors are therefore called element or type selectors, since they select based on an element. There are other types of selectors, namely:

  • class selectors - select elements based on their class attribute
  • ID selectors - select elements based on their id attribute
  • Pseudo-class and pseudo-element selectors - rules that style certain states of an element, e.g. on mouse hover, on click
There are a few other, but we won't look into them right now.


Finally, it also makes sense to somehow combine these selectors to allow even greater precision. For example, you might want to select all h1 elements which are children of an element with some distinct ID. For such purposes, CSS uses combinators. There are a few of these:

  • Selector list A, B - apply the rule to both A and B
  • Adjacent sibling selector A + B - apply the rule to an element B which is a direct sibling of A
  • Child selectors A > B - apply the rule to an element B which is a direct child of A
  • Descendant selectors A B - apply the rile to an element B which is a descendant of A
